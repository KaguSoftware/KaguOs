# Product

## Register

product

## Platform

web

## Users
The 8 people of Kagu, an Istanbul software studio: four builders (Work section), all eight
learners (Learn), two managers (Management), everyone on the Debug board. They use KaguOs daily at
their desks between coding sessions — quick check-ins to grab a debug task, tick a sprint goal, or
skim project status — plus a few deeper sessions per week for ledger entries, contracts, and sprint
planning. Primary audience: the whole team on the boards. Secondary: Parsa and one other manager
wearing the admin/management hats.

## Product Purpose
KaguOs is Kagu's single internal system, replacing scattered tools (a Google Sheets task board,
ad-hoc records). Five membership-gated sections behind one login: Work (projects + ideas), Learn
(learning sprints with per-person progress), Management (multi-currency ledger + contracts), Debug
(claim-a-task board), Marketing (shell for now). Success: the old sheet retires, tasks get claimed
and finished visibly, sprint progress is legible at a glance, and the company's records live in one
trusted place.

## Positioning
Everything Kagu runs on, in one quiet place — see what needs doing, claim it, get on with it.

## Brand Personality
Quiet and precise. Kagu's public line is "we build what operates, not what demos" — the internal
tool holds itself to the same standard. Dark-first, calm, typography-led, information-dense without
noise. Confidence through restraint; the tool should feel like a well-made instrument, not a demo.

## Anti-references
- SaaS dashboard clichés: gradient hero metrics, glassmorphism, identical icon-card grids,
  decorative illustration.
- Startup-landing energy: confetti, mascots, gamification badges.
- Enterprise bloat: configuration mazes, ten-click flows for one-click jobs.

## Design Principles
1. The task is the interface — every screen answers "what needs my attention?" first.
2. One-click primitives — claim, tick, and state changes are single actions, never forms.
3. Calm density — show plenty, decorate nothing.
4. Membership is invisible until it matters — you simply don't see sections you're not in.
5. Trustworthy records — money and contracts read like a well-kept ledger, not a widget.

## Accessibility & Inclusion
WCAG AA contrast (≥4.5:1 body text) on the dark theme. Full keyboard operability for claim/tick
flows. `prefers-reduced-motion` honored everywhere. English UI for a non-native-English team:
plain words, no idioms.
